# DOX: modules/

## Purpose

UUID-addressed modular packages for Danwa. Each module contains a `manifest.json` and domain-specific content (agent profiles, bundles, LLM configs, workflows, language packs).

## Ownership

- **Agent Argumentation Patterns**: `modules/agent-argumentation-patterns/` — 12 philosophical argumentation styles
- **Agent Bundles**: `modules/agent-bundles/` — 72 pre-configured agent role combinations
- **Agent Cores**: `modules/agent-cores/` — 34 individual agent role definitions
- **Agent Tone Profiles**: `modules/agent-tone-profiles/` — 3 tone profiles
- **Kitsune Assistant**: `modules/kitsune-assistant/` — AI assistant module
- **Language Packs**: `modules/lang-de/` — German language pack
- **LLM Profiles**: `modules/llm-profiles/` — 21 LLM provider configurations
- **Prompt Modifiers**: `modules/prompt-modifiers/` — prompt modification rules
- **Workflows**: `modules/workflows/` — 8 workflow definitions

## Local Contracts

- Each module directory is named by UUID (e.g., `ac-0e5e3777-...`)
- Every module must have a valid `manifest.json` matching `schemas/module-manifest.json`
- Module registry is in `index.json` (generated by `scripts/generate_index.py`)

## Work Guidance

- Never edit module directories manually; use migration scripts
- Regenerate `index.json` after adding/removing modules
- Follow UUID naming convention for new modules

## Verification

- Validate manifests against `schemas/module-manifest.json`
- Run `python3 scripts/generate_index.py` to verify index generation
