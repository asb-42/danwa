## Definition: Was sind Agenten in Danwa?

In Danwa gibt es momentan zwei Typen von Agenten:

1. Eigene (interne) Agenten:

- Werden vollständig im System erstellt und verwaltet.

2. Externe Agenten:

- Werden über A2A eingebunden.
- Können ihre eigene Persona mitbrigen (Regelfall), können aber auch im System modifiziert werden.

## Komponenten interner Agenten

Ein **interner Agent** besteht aus folgenden Komponenten:

### 1. Agent Core

- war zuvor: Role Type
- Funktionale Kernrolle des Agenten: Was macht der Agent

- Vordefinierte Agent-Cores:
a) strategist: Strukturierung, Themenextraktion, Rahmen setzen
b) critic: Schwachstellenidentifikation, Devil’s Advocate
c) optimizer: Synthese, Verfeinerung, Verbesserung
d) moderator: Konsensbildung, Abschlussbewertung, Zeitmanagement
e) fact-checker: Faktische Verifizierung, Quellenprüfung
f) analyst: Tiefgehende Datenanalyse, Mustererkennung
g) creative: Kreative Ideenfindung, Perspektivwechsel, Brainstorming
h) expert-reviewer: Domänen-spezifische Expertenbewertung

Formatorisch modifizierte Rollen:
i) interviewer: Strukturierte Interviews, gezieltes Nachfragen (formatorische Modifikation von strategist oder analyst)
j) interviewee: Interviewpartner in strukturiertem Interview, antwortet auf Fragen von interviewer
k) advocate: Einseitige Verteidigung einer Position; im Streitgespräch: These (formatorische Modifikation von strategist)
l) adversary: Aktive Konfrontation, Widerlegung; im Streitgespräch: Antithese (formatorische Modifikation von critic)
m) mediator: Vermittlung, Kompromissfindung (formatorische Modifikation von moderator oder optimizer)
n) referee: Durchsetzen von Regeln, Bewerten von Fairness (formatorische Modifikation von moderator)
o) facilitator: Prozessförderung, ohne inhaltliche Bewertung (formatorische Modifikation von moderator)
p) devils-advocate: Systematisches Infragestellen ohne direkte Konfrontation (formatorische Modifikation von critic)

### 2. Argumentationsmuster (argumentation pattern)

- war zuvor: war: Role Definition
- Wie argumentiert der Agent methodisch? Mit welcher Methodik/Methode argumentiert der Agent?

- Vordefinierte Argumentationsmuster:
a) kantian - Philosophische/weltanschauliche Grundlage: Kategorischer Imperativ, Pflichtethik
b) hegelian -  Philosophische/weltanschauliche Grundlage: Hegelsche Dialektik (These–Antithese–Synthese)
c) utilitarian - Philosophische/weltanschauliche Grundlage: Utilitarismus, größtmögliches Glück 
d) stoic - Philosophische/weltanschauliche Grundlage: Stoische Tugendethik, Gleichmut 
e) aristotelian - Philosophische/weltanschauliche Grundlage: Aristotelische Tugendethik, Golden Mean
f) socratic - Philosophische/weltanschauliche Grundlage: Sokratische Methode, maieutisches Fragen
g) steiner - Philosophische/weltanschauliche Grundlage: Anthroposophie; Analyse nach dreigliedriger Wesensstruktur des Menschen (Denken, Fühlen und Wollen) bzw.  Berücksichtige der dreigliedrigen Sozialordnung (Rechtsleben, Geistesleben, Wirtschaftsleben).

### 3. Tone Profile

- Charakterisiert die Engagiertheit und Stimmung des debattierenden Agenten
- Tone Profiles ändern die gesamte Atmosphäre einer Debatte, z.B. hitzig-emotionales Streitgespräch vs. akademisch-abgehobene Debatte

- Vordefinierte Tone Profiles
a) neutral - Ausgewogen, sachlich, ohne emotionale Färbung (Default)
b) academic - Sachlich, analytisch, zitierungsfreudig
c) conversational - Natürlich, umgangssprachlich, freundlich
d) socratic - Fragenstellerisch, entdeckend, nachforschend
e) heated - Emotional, fordernd, polemisch
f) aggressive - Aggressiv, evtl. beleidigend (Steigerung von heated)


### 4. Prompt Modifier

- war zuvor: Prompt Variant
- Feineinstellungen zur Präsentation des Beitrags
- Beispiel: "nur Fließtext, keine Tabellen, keine Aufzählungen" (erforderlich für Audio-Ausgabe/Podcast)

## Komponenten externer Agenten

- Externe Agenten bringen typischerweise ihre Persona mit, können aber optional durch dieselben Komponenten modifiziert werden wie die internen Agenten.
- Wenn externe Agenten Teil eines Agent-Bundles sind, wird der finale Prompt nicht an eine LLM weitergereicht, sondern via A2A an den Agent Provider übermittelt.

## Regeln für alle Komponenten

1) Jede der Agent-Komponenten enthält

- ein Title-Feld
- ein Description-Feld
- ein Textfeld mit einem Prompt-Stub.

Diese Felder können über das UI bearbeitet werden.

Zusätzlich enthalten die Agent-Komponenten Verwaltungs- und Metainformationen (z.B. eindeutige ID), die nicht im UI bearbeitet werden können.

2) Der Prompt-Stub ist SSOT en und kann über UI übersetzt & bearbeitet & exportiert werden, sobald er in die DB geladen wurde.

## Agent-Bundles

- Die Prompts-Stubs aus den Agent-Kompnenten werden zu einem finalen Prompt zusammengefügt und ergeben dann zusammen mit dem LLM-Profil ein Agent-Bundle.
- Agent-Bundles sind die Bestandteile, die in einem Workflow in Erscheinung treten. In einem Workflow gibt es keinen alleinstehenden "Agent Core" und keine "Prompt Variants", nur Agent Bundles.

### Speichern von Agenten-Komponenten und Bundles

- Alle Komponenten von Agenten sind modularisiert.
- Speicherart SSOT: Dateisystem (versionierbar, externes Repository zum Verwalten und Verteilen)
- Sprache: Englisch
- Import aus Modul in Datenbank
- In Datenbank: Übersetzung in die UI-Sprachen
- Standardverhalten: UI-Sprache en -> Debattensprache en; UI-Sprache de -> Debattensprache de

### Agent-Bundles und Module

Alle Komponenten von Agenten sind im UI
- ansehbar,
- editierbar,
- duplizierbar,
- übersetzbar,
- exportierbar.

Beim Exportieren einer Komponente wird ein Modul erzeugt.

Module folgen schemas/module-manifest.json
